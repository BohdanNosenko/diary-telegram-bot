# Vlog Journal — Fresh Machine Deployment & Setup Guide

This guide provides step-by-step instructions for deploying `vlog-journal` on a fresh Linux host or WSL2 environment with NVIDIA GPU acceleration.

---

## 🛠️ 1. Hardware & System Prerequisites

### Minimum Requirements:
- **OS**: Linux (Ubuntu 22.04 LTS or newer) on bare metal or WSL2 (Windows 11 recommended).
- **CPU**: 4+ cores.
- **GPU**: NVIDIA GPU with at least **8 GB VRAM** (e.g. RTX 3060, RTX 4070, or higher).
- **RAM**: 16 GB system memory.
- **Storage**: 50 GB free disk space.

---

## 🐧 2. System Setup & NVIDIA Drivers

### A. WSL2 Setup (Windows Hosts)
1. Install Ubuntu on WSL2 from PowerShell:
   ```powershell
   wsl --install -d Ubuntu
   ```
2. Verify NVIDIA CUDA passthrough inside WSL2:
   ```bash
   nvidia-smi
   ```

### B. Install Nix & direnv
Install Nix package manager in single-user or multi-user mode:

```bash
sh <(curl -L https://nixos.org/nix/install) --daemon
```

Enable Flakes in `~/.config/nix/nix.conf`:
```text
experimental-features = nix-command flakes
```

Install `direnv` and hook it to bash:
```bash
sudo apt update && sudo apt install -y direnv
echo 'eval "$(direnv hook bash)"' >> ~/.bashrc
source ~/.bashrc
```

## 🐳 3. Telegram Local Bot API Server (Docker)

To bypass Telegram's default 50 MB bot file download limit, run the official Telegram Bot API server locally via Docker Compose.

1. Obtain your `TELEGRAM_API_ID` and `TELEGRAM_API_HASH` from [my.telegram.org](https://my.telegram.org).
2. Configure `docker-compose.yml`:
   ```yaml
   services:
     telegram-bot-api:
       image: aiogram/telegram-bot-api:latest
       container_name: telegram_local_server
       environment:
         TELEGRAM_API_ID: "${TELEGRAM_API_ID}"
         TELEGRAM_API_HASH: "${TELEGRAM_API_HASH}"
       ports:
         - "8081:8081"
       volumes:
         - telegram-bot-api-data:/var/lib/telegram-bot-api
       restart: always

   volumes:
     telegram-bot-api-data:
   ```
3. Start the container:
   ```bash
   docker compose up -d
   ```

## 🔑 4. Third-Party Credentials & Authentication

### A. HuggingFace Token (Pyannote Diarization)
1. Accept user conditions for model access:
   - [pyannote/speaker-diarization-3.1](https://huggingface.co/pyannote/speaker-diarization-3.1)
   - [pyannote/segmentation-3.0](https://huggingface.co/pyannote/segmentation-3.0)
2. Generate an Access Token under HuggingFace Settings -> Access Tokens.
3. Save token in `.env`:
   ```env
   HF_TOKEN="hf_..."
   ```

### B. Rclone Configuration (Google Drive Backups)
1. Configure Rclone remote named `gdrive`:
   ```bash
   rclone config
   # Follow prompts to set up Google Drive remote named 'gdrive'
   ```
2. Verify remote connectivity:
   ```bash
   rclone lsjson gdrive:
   ```

## ⚙️ 5. Application Configuration

Clone the repository and allow direnv:

```bash
git clone https://github.com/BohdanNosenko/diary-telegram-bot.git vlog-journal
cd vlog-journal
direnv allow
```

Copy and edit configuration templates:

```bash
cp .env.example .env
cp config.example.toml config.toml
```

Ensure `.env` contains:
```env
TELEGRAM_BOT_TOKEN="your-bot-token"
ALLOWED_USER_IDS="123456789"
HF_TOKEN="hf_your_token"
BACKUP_ENCRYPTION_PASSPHRASE="your-secure-passphrase"
```

Ensure `config.toml` paths match your environment:
```toml
[app]
vault_name = "PersonalVault"
vault_path = "/home/user/documents/PersonalVault"
vlogs_relative_path = "Journal/Vlogs"
media_relative_path = "Attachments/Vlogs"
```

## 🚀 6. Running as a Systemd Service

Create a systemd user service file at `~/.config/systemd/user/vlog-journal.service`:

```ini
[Unit]
Description=Vlog Journal Telegram Bot
After=network.target docker.service

[Service]
Type=simple
WorkingDirectory=/home/user/vlog-journal
ExecStart=/home/user/.nix-profile/bin/nix develop --command uv run vlog-journal
Restart=always
RestartSec=10

[Install]
WantedBy=default.target
```

Enable and start the service:

```bash
systemctl --user daemon-reload
systemctl --user enable vlog-journal
systemctl --user start vlog-journal
```

Check service status and logs:

```bash
systemctl --user status vlog-journal
journalctl --user -u vlog-journal -f
```

## ❓ 7. Troubleshooting Common Issues

| Symptom | Cause | Solution |
|---|---|---|
| `CUDA out of memory` | GPU VRAM exhausted by concurrent models | Ensure Whisper and Pyannote models unload after use. Close background GPU apps. |
| `Telegram API 401 Unauthorized` | Invalid `TELEGRAM_BOT_TOKEN` | Verify bot token generated by `@BotFather` in `.env`. |
| `rclone: command not found` | Rclone not installed | Enter Nix dev shell (`nix develop`) or install `rclone`. |
| `HuggingFace model 403 Forbidden` | License terms not accepted for Pyannote | Visit Pyannote HuggingFace model pages and accept user terms. |
| `Ollama connection refused` | Ollama server not running | Run `ollama serve &` and verify `http://localhost:11434/api/tags`. |
